** TODO [#B] Fix HashSet rebuilt on every fix_expression call :refactor:perf:
:PROPERTIES:
:CUSTOM_ID: 7
:CREATED: [2026-02-24 Tue]
:END:
In ~adapter.rs~, ~esbmcfixes::fix_expression~ reconstructs a ~HashSet<String>~ of known expression IDs on every invocation. Since ~fix_expression~ is called recursively for every node in every instruction in every function, this causes a heap allocation and N string copies per node.

Replace with a compile-time static set using the ~phf~ crate:
#+begin_src rust
static EXPRESSIONS: phf::Set<&'static str> = phf::phf_set! {
    "if", "member", "typecast", ...
};
#+end_src
Or at minimum a ~OnceLock<HashSet<&'static str>>~ so it is built once.

** TODO [#C] Visitor pattern for tree transformations              :refactor:
:PROPERTIES:
:CUSTOM_ID: 11
:CREATED: [2026-02-24 Tue]
:END:
~fix_type~ and ~fix_expression~ in ~adapter.rs~ both implement the same recursive traversal over ~subt~, ~named_subt~, and ~comments~. This traversal logic is duplicated across both functions.

Extract an ~IrepVisitor~ trait with a default ~walk~ method that handles traversal, so each transformation only implements the per-node logic:
#+begin_src rust
trait IrepVisitor {
    fn visit(&mut self, irep: &mut Irept);

    fn walk(&mut self, irep: &mut Irept) {
        self.visit(irep);
        for v in &mut irep.subt { self.walk(v); }
        for (_, v) in &mut irep.named_subt { self.walk(v); }
        for (_, v) in &mut irep.comments { self.walk(v); }
    }
}
#+end_src

** TODO [#C] Cache hash value on Irept                        :refactor:perf:
:PROPERTIES:
:CUSTOM_ID: 12
:CREATED: [2026-02-24 Tue]
:END:
~ByteWriter~ uses ~irep_container: HashMap<Irept, u32>~ as a deduplication cache. Each lookup hashes the full ~Irept~ tree recursively. Since ~Irept~ is immutable after construction, the hash can be computed once and stored:
#+begin_src rust
pub struct Irept {
    pub id: String,
    pub subt: Vec<Irept>,
    pub named_subt: HashMap<String, Irept>,
    pub comments: HashMap<String, Irept>,
    cached_hash: u64,
}
#+end_src
~Hash::hash~ would then feed only ~self.cached_hash~ into the hasher — O(1) regardless of tree depth. Best implemented together with #10 (~Rc<Irept>~).

** DONE [#C] Rc<Irept> for structural sharing                 :refactor:perf:
:PROPERTIES:
:CUSTOM_ID: 10
:CREATED: [2026-02-24 Tue]
:END:
~Irept~ is cloned heavily throughout ~adapter.rs~ (e.g. ~sym.stype.clone()~, ~cache[...].clone()~, ~*self = cache[...].clone()~). Each clone deep-copies the entire subtree.

Wrap child nodes in ~Rc<Irept>~:
#+begin_src rust
pub struct Irept {
    pub id: String,
    pub subt: Vec<Rc<Irept>>,
    pub named_subt: HashMap<String, Rc<Irept>>,
    pub comments: HashMap<String, Rc<Irept>>,
}
#+end_src
~.clone()~ on an ~Rc~ is O(1) — only the reference count is incremented. This directly addresses the ~TODO: This should be references and not copies!~ comment in ~irep.rs:7~.
